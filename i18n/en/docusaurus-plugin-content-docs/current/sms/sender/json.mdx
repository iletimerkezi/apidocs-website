# JSON Post API
If you want to retrieve the list of approved headers defined in your account using the POST method of the HTTP protocol in JSON format, you can follow the guidelines below.

## İstek yapılacak adres

<RequestURL type="POST" url="https://api.iletimerkezi.com/v1/get-sender/json" />

## Request URL (Body)

```json
{
  "request": {
    "authentication": {
      "key": "",
      "hash": ""
    }
  }
}
```

### Tanımlar

The request to be made to the server must be wrapped in a <Badge>request</Badge> tag. The definitions of the sub-tags of this tag are as follows:

<AccessDescription />

## Server Response

```json
{
  "response": {
    "status": {
      "code": "",
      "message": ""
    },
    "senders": {
      "sender": [
        "APITEST",
        "eMarka"
      ]
    }
  }
}
```

### Definitions

The response from the server is always wrapped in a <Badge>response</Badge> tag. The definitions of the sub-tags of this tag are as follows:

<ResponseStatus />

#### senders
If the operation is completed successfully, it contains the information about the created order. If the operation fails, this tag is not returned. The sub-tags are as follows:

+ **sender:** Contains the approved header information.

## Sample Request
```json
{
  "request": {
    "authentication": {
      "key": "507caf2e1fcdb5eea9786332ca2d8785",
      "hash": "0db4e316db72c519ba08121985f6ddf479809053d555c"
    }
  }
}
```

## Sample Successful Response
```json
{
  "response": {
    "status": {
      "code": 200,
      "message": "Operation successful"
    },
    "senders": {
      "sender": [
        "eMarka"
      ]
    }
  }
}
```

## Sample Error Response
```json
{
  "response": {
    "status": {
      "code": 401,
      "message": "Membership information is incorrect."
    }
  }
}
```
