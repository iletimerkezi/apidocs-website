# JSON Post API
<AccessWarning />
To retrieve the information of a contact in your address book using the POST method of the HTTP protocol in JSON format, you can follow the instructions below.

## Request URL

<RequestURL type="POST" url="https://api.iletimerkezi.com/v1/get-contact/json" />

## Request Body

```json
{
  "request": {
    "authentication": {
      "key": "",
      "hash": ""
    },
    "contact": {
      "id": ""
    }
  }
}
```

### Definitions

The request sent to the server must be wrapped in the <Badge>request</Badge> tag. The definitions of its sub-tags are as follows:

<AccessDescription />

#### contact
Contains the information of the contact whose information you want. The sub-tag is as follows.

+ **id:** Represents the unique numeric value of the contact.

## Server Response

```json
{
  "response": {
    "status": {
      "code": "",
      "message": ""
    },
    "contact": {
      "id": "",
      "name": "",
      "surname": "",
      "email": "",
      "gsm": "",
      "custom_fields": [
        {
          "id": "",
          "value": ""
        }
      ],
      "groups": [5]
    }
  }
}
```

### Definitions

The response from the server is always wrapped in the <Badge>response</Badge> tag. The definitions of its sub-tags are as follows:

<ResponseStatus />

#### contact
Contains the information of the contact. It does not repeat. The sub-tags are as follows.

+ **id:** Represents the unique numeric value of the contact.
+ **name:** Represents the contact's name.
+ **surname:** Represents the contact's surname.
+ **email:** Represents the contact's email address.
+ **gsm:** Represents the contact's mobile phone number.

+ #### custom_fields
Represents the custom fields defined for the contact. The sub-tags are as follows:

  + **id:** Represents the unique numeric value of the custom field.
  + **value:** Represents the value stored for the contact in the custom field.

+ #### groups
Contains the unique numeric values representing the groups to which the contact belongs.
